## 4. 📝 사용자 요구사항

### 기능 요약

캘린더 뷰에서 반복 일정을 아이콘을 넣어 구분하여 표시한다.

### 상세 요구사항

- 캘린더 뷰에서 반복 일정(repeat.type !== 'none')을 일반 일정과 구분하여 표시
- 아이콘을 통해 시각적으로 반복 일정임을 알 수 있도록 함
- 기존 UI 컴포넌트 활용

### 제약 사항

1. 이미 존재하는 UI를 사용
2. server.js 문서를 확인해 API 파악 후 작업
3. 추가/수정한 반복 일정이 캘린더에 잘 표시되는지 확인
4. 추가/수정한 반복 일정이 일정 목록에 잘 표시되는지 확인
5. 추가/수정 후 이벤트 로딩 에러가 발생하지 않는지 확인
6. 각 에이전트는 작업 전 guide 문서 숙지, 작업 후 checklist로 셀프체크
7. Zeus는 guide 문서 숙지, 각 단계 완료 시 커밋 (한국어)
8. 코드 작성 시 의존성 최소화 (date-fns 금지)
9. Apollo의 리팩토링 범위는 Hermes 코드로만 제한
10. 최대한 기존 코드 활용, 기존 로직 재사용
